# Phase 05 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | `b5J5C8tpo` (`radix-nova`, from `components.json`) |
| Component library | radix |
| Icon library | phosphor |
| Font | `Noto Sans` via `next/font` |

Operations dashboard must preserve the existing admin shell and schedule-detail workflow:
- Render the dashboard as the default admin landing surface at `/admin`, not as a detached report page.
- Keep the dashboard summary-first. Cards show operational signals only; detailed correction stays in the existing schedule detail route.
- Use schedule cards as the primary summary unit, not a worker-first table or analytics grid.
- Split the page into `Today` and `Upcoming` sections so immediate work is visually dominant while nearby upcoming schedules remain visible.
- Use official shadcn primitives only for this phase. No third-party registries are allowed.

Visual priority contract:
- Page headline and lightweight triage summary first.
- `Today` schedule cards second.
- `Upcoming` schedule cards third.
- Within each card, highest-priority anomaly first, staffing and attendance summaries second, supporting metadata last.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Badge icon gaps, inline metadata spacing |
| sm | 8px | Label-to-value spacing, compact card row gaps |
| md | 16px | Default card padding, stacked content spacing |
| lg | 24px | Card-to-card spacing, section padding |
| xl | 32px | Page section gaps, header-to-grid separation |
| 2xl | 48px | Major separation between `Today` and `Upcoming` sections |
| 3xl | 64px | Desktop page outer spacing in the admin shell |

Exceptions: `44px` minimum target height for card-level primary links or buttons; `20px` square status icons are allowed inside badges and summary rows.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Additional contract:
- Use `Display` once at the top of the page for the triage headline or highest-level operations summary.
- Use `Heading` for section titles such as `Today`, `Upcoming`, and card titles that need emphasis.
- Use `Label` for anomaly badges, metric labels, and compact metadata keys.
- Do not introduce additional weights beyond `400` and `600` in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / `oklch(1 0 0)` | Page background, primary card surface, default dashboard canvas |
| Secondary (30%) | `var(--secondary)` / `oklch(0.967 0.001 286.375)` | Triage summary panel, secondary card bands, grouped fact rows, section containers |
| Accent (10%) | `var(--primary)` / `oklch(0.5 0.134 242.749)` | Page-level triage summary number, highest-priority anomaly badge, `Review schedule` CTA, active focus states |
| Destructive | `var(--destructive)` / `oklch(0.577 0.245 27.325)` | Not used by default in this phase because no destructive action is introduced |

Accent reserved for: page-level triage metric, one highest-priority anomaly badge per schedule card, the primary drill-down CTA, and current-focus interactive states. Do not use accent for every badge, every link, or passive metadata.

Semantic status usage:
- `Unfilled slots` is the strongest visual anomaly and may use accent-filled badge treatment.
- `Missing check-ins` uses accent-outline or strong text emphasis when it is the top unresolved issue for a card.
- `Late` uses muted or secondary emphasis unless it is the only unresolved anomaly.
- Healthy cards use dominant and secondary surfaces with muted metadata and no accent-filled alert treatment.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Review schedule |
| Empty state heading | No schedules need attention right now |
| Empty state body | There are no schedules for today or the nearby upcoming window. Check back later or open the schedule list to review future staffing. |
| Error state | We could not load the operations dashboard. Refresh the page, and if the problem continues, open the schedule list or a schedule detail page directly. |
| Destructive confirmation | Not applicable: this phase introduces no destructive action. |

Additional copy rules:
- Card titles should lead with the schedule name or role grouping context, then show the schedule date and start time nearby.
- Anomaly labels must stay short and ordered by priority: `Unfilled slots`, `Missing check-ins`, `Late arrivals`.
- Summary labels should be noun-first and scannable: `Applicants`, `Confirmed`, `Checked in`, `Starts at`.
- If a card has no anomaly, use reassuring but neutral copy such as `On track` rather than celebratory language.
- Time-sensitive copy must show exact local times, not vague phrases like `soon`.
