## Interaction Contract

- Admin schedule detail uses a two-panel layout on desktop and a single-column stack on mobile. The left panel holds applicants and slot assignment controls; the right panel holds schedule facts, slot fill summary, and the sticky action area.
- The primary focal point on the admin screen is the slot fill summary at the top of the right panel. It should draw first glance before the applicant assignment table and the sticky confirm area.
- Visual hierarchy for the admin screen is fixed as: slot fill summary -> applicant assignment controls -> sticky confirm action.
- Applicant state uses exactly three labels: `미배정`, `배정안 저장됨`, `확정됨`. Do not introduce inbox-style or message-style terminology.
- Draft save is non-blocking and must preserve the current editing surface. After save, show an inline success notice near the sticky action area; do not redirect.
- Final confirm is a separate modal-confirmed action. The dialog must summarize filled slots, unfilled slots, and the worker-visible consequence before submit.
- Worker-facing preview shows confirmed work only. The page order is `total pay`, `calculation basis`, `confirmed assignments list`; draft state is never rendered on worker routes.
- Pay breakdown uses one summary card per assignment with four fixed facts: date/time, role, hourly rate, overtime applied yes/no. Total pay appears once at page top, not repeated per row.
