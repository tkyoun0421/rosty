# Phase 04 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | `b5J5C8tpo` (`radix-nova`, from `components.json`) |
| Component library | radix |
| Icon library | phosphor |
| Font | `Noto Sans` via `next/font` |

Attendance check-in must preserve the existing split between worker and admin flows:
- Worker check-in extends the confirmed assignment experience, not a separate discovery page.
- Admin attendance review extends the schedule detail experience, not a dashboard.
- Use official shadcn primitives only for this phase. No third-party registries are allowed.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline badge gaps, icon spacing, helper text offsets |
| sm | 8px | Form label to field spacing, metadata row gaps |
| md | 16px | Default card padding, stacked content spacing |
| lg | 24px | Section padding, card-to-card spacing |
| xl | 32px | Major panel gaps, page section separation |
| 2xl | 48px | Worker page breathing room between summary and check-in panels |
| 3xl | 64px | Desktop page outer spacing when a full-page shell is used |

Exceptions: `44px` minimum touch target height for the worker check-in CTA and any icon-only controls.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Additional contract:
- Use `Display` once per page for the worker check-in status or admin attendance headline metric.
- Use `Heading` for section titles such as `Check-in status`, `Location confirmation`, and `Attendance review`.
- Do not introduce additional weights beyond `400` and `600` in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / `oklch(1 0 0)` | App background, worker page canvas, admin detail canvas |
| Secondary (30%) | `var(--secondary)` / `oklch(0.967 0.001 286.375)` | Cards, grouped schedule facts, muted admin side panels |
| Accent (10%) | `var(--primary)` / `oklch(0.5 0.134 242.749)` | Primary check-in CTA, open-window badge, active attendance summary chip, success-forward progress emphasis |
| Destructive | `var(--destructive)` / `oklch(0.577 0.245 27.325)` | Submission warning copy, blocked-location errors, irreversible action confirmation only |

Accent reserved for: `Check in now` button, `Check-in open` badge, worker success state highlight, admin late/on-time status summary chips, and current-focus interactive element states. Do not use accent for all links, all borders, or passive metadata.

Semantic status usage:
- On-time attendance rows use dominant surface plus accent text/badge treatment.
- Late attendance rows use dominant surface plus destructive-adjacent text emphasis, but keep the row readable rather than fully red.
- Pending or not-yet-open states use `secondary` or `muted`, never accent.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Check in now |
| Empty state heading | No confirmed shift ready for check-in |
| Empty state body | Your confirmed assignment will appear here when check-in opens. If you already have a shift, wait for the check-in window or contact an admin. |
| Error state | We could not complete check-in. Confirm location access, move inside the venue check-in radius, and try again before the window closes. |
| Destructive confirmation | Submit check-in: Submit this attendance record now? You can only check in once and cannot edit it later. |

Additional copy rules:
- Worker helper copy must always explain why check-in is unavailable: `not open yet`, `outside allowed radius`, or `already submitted`.
- Admin status labels must stay short and scannable: `Checked in`, `Late`, `Not checked in`, `Not open yet`.
- Time-based messages must show the exact opening time when known.
